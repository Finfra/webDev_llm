# Cursor 사용 실습

## 1. Cursor 소개
Cursor는 AI 기반 코드 편집기로, Visual Studio Code를 기반으로 하며 강력한 AI 기능이 통합되어 있음.  기존 코드 편집기와 달리 Cursor는 코드의 의미를 이해하고 맥락에 맞는 도움을 제공할 수 있음.  개발자의 생산성을 높이고 코딩 경험을 혁신적으로 개선하기 위해 설계되었음. 

### 주요 특징
- **AI 기반 코드 완성**: 다음 코드 라인을 예측하고 자동 완성 기능 제공
- **코드 자동 생성**: 자연어 지시사항을 바탕으로 전체 코드 블록 생성
- **문맥 인식 기능**: 전체 코드베이스를 이해하고 관련 코드를 참조
- **다양한 프로그래밍 언어 지원**: Python, JavaScript, Java, C++ 등 지원
- **VS Code와의 호환성**: 익숙한 VS Code 인터페이스와 확장 프로그램 사용 가능

## 2. Cursor 설치 방법
Cursor는 Windows, macOS 및 Linux에서 사용할 수 있음.  설치 과정은 간단함. 

### 설치 단계
1. [Cursor 공식 웹사이트](https://cursor.com)에 접속함. 
2. 화면 상단의 [Download] 버튼을 클릭함. 
3. 운영체제에 맞는 설치 파일을 다운로드함. 
4. 다운로드한 설치 파일을 실행함. 
5. 설치 과정에서 언어 선택에서 '한국어'를 입력하고, 원하는 설정을 선택함. 
6. 설치가 완료되면 Cursor를 실행함. 

### 초기 설정
- 첫 실행 시 VS Code 설정을 가져올지 묻는 메시지가 표시됨. 
- 기존 VS Code 사용자는 테마, 키 바인딩, 확장 프로그램 등을 가져오는 것이 유리함. 
- AI 기능을 사용하기 위해 로그인이 필요할 수 있음. 
- 데이터 수집 동의 여부를 선택할 수 있음.  원하지 않으면 'Privacy Mode'를 선택하세요.

## 3. 주요 기능 살펴보기
Cursor의 핵심 AI 기능들은 코딩 경험을 혁신적으로 개선함. 

### Cursor Tab (코드 완성)
- 코드 작성 시 자동으로 다음 라인을 제안함. 
- 단순한 라인 완성뿐만 아니라 복잡한 다중 라인 코드 스니펫도 제안함. 
- Tab 키를 눌러 AI 제안을 수락할 수 있음. 

### AI 채팅 (Ctrl+L 또는 Cmd+L)
- 코드에 대한 질문이나 설명 요청을 자연어로 할 수 있음. 
- 코드베이스에 관한 질문에 답변하고 코드 개선 제안을 제공함. 
- 선택한 코드를 사용하여 자동으로 컨텍스트를 인식함. 
- 'Apply' 버튼으로 코드 변경사항을 바로 적용할 수 있음. 

### Cmd+K (코드 편집 및 생성)
- 기존 코드를 수정하거나 새 코드를 생성할 수 있음. 
- 코드를 선택하고 Cmd+K(또는 Ctrl+K)를 누른 후 자연어로 지시사항을 입력함. 
- AI가 지시사항을 해석하여 코드를 수정하거나 생성함. 

### Composer (Cmd+I 또는 Ctrl+I)
- 더 복잡한 작업이나 다중 파일 변경을 위한 도구임. 
- 큰 작업을 단계별로 나누어 진행할 수 있음. 
- 코드베이스 탐색, 새 기능 작성, 기존 코드 수정을 지원함. 

### @기능 (멘션 기능)
- @파일명으로 로컬 파일 내용을 참조할 수 있음. 
- @웹주소로 웹 페이지 내용을 불러올 수 있음. 
- 코드 스니펫, 문서 내용, 이미지 등 다양한 유형의 콘텐츠를 멘션하여 활용할 수 있음. 

### 터미널 통합
- 내장 터미널에서 Ctrl+K를 사용하여 자연어로 터미널 명령어를 생성할 수 있음. 
- 명령어 실행 결과를 바탕으로 AI가 다음 단계를 제안할 수 있음. 

## 4. 코드 자동 생성과 수정
Cursor를 사용하면 코드를 처음부터 작성하거나 기존 코드를 쉽게 수정할 수 있음. 

### 코드 생성 방법
1. **채팅을 통한 생성**: Ctrl+L을 눌러 채팅 패널을 열고 원하는 코드에 대한 설명을 입력함. 
   ```
   "Python으로 간단한 웹 서버를 만들어주세요."
   ```

2. **인라인 편집을 통한 생성**: 주석으로 원하는 기능을 설명하고 선택한 후 Ctrl+K를 누릅니다.
   ```python
   # 사용자 입력을 받아 팩토리얼을 계산하는 함수 작성
   ```

3. **Composer를 통한 생성**: Ctrl+I를 눌러 Composer를 열고 더 복잡한 프로젝트나 다중 파일 코드를 생성함. 
   ```
   "사용자 로그인과 간단한 게시판 기능이 있는 웹 애플리케이션을 만들어주세요."
   ```

### 코드 수정 요청
- 코드를 선택하고 Ctrl+K를 눌러 수정사항을 요청함. 
  ```
  "이 함수에 에러 처리를 추가하고 성능을 최적화해주세요."
  ```
- 채팅에서 코드를 참조하고 수정을 요청할 수도 있음. 
  ```
  "위 코드에서 for 루프 대신 list comprehension을 사용하도록 수정해주세요."
  ```

### 효과적인 프롬프트 작성법
- 구체적이고 명확한 지시사항을 제공함. 
- 원하는 결과물의 예시나 형식을 언급함. 
- 단계별로 요청을 나누어 복잡한 작업을 수행함. 
- 프로그래밍 언어나 프레임워크를 명시함. 
- 코드 스타일이나 특정 패턴 사용을 요청할 수 있음. 

## 5. 문맥 이해 능력과 활용
Cursor는 단순히 코드 조각을 생성하는 것을 넘어, 전체 코드베이스의 문맥을 이해하고 활용할 수 있음. 

### 코드베이스 이해
- Cursor는 프로젝트의 모든 파일을 분석하여 전체 구조를 파악함. 
- 파일 간의 관계, 의존성, 함수 호출 등을 인식함. 
- @Codebase 명령어를 사용하여 전체 코드베이스에 관한 질문을 할 수 있음. 
  ```
  "@Codebase 이 프로젝트의 핵심 기능은 무엇이며 어떻게 구현되어 있나요?"
  ```

### 문맥 기반 코드 제안
- 현재 작업 중인 파일의 문맥을 고려한 코드를 제안함. 
- 프로젝트에서 사용 중인 변수, 함수, 클래스 등을 인식하고 활용함. 
- 코드의 일관성을 유지하며 새로운 코드를 생성함. 

### 다중 파일 편집
- Composer를 사용하여 여러 파일에 걸친 변경사항을 한 번에 적용할 수 있음. 
- 관련된 여러 파일(예: HTML, CSS, JavaScript)을 함께 수정할 수 있음. 
- 대규모 리팩토링이나 새 기능 추가 시 유용함. 

### 커스텀 문맥 설정
- .cursorrules 파일을 생성하여 프로젝트별 AI 규칙을 설정할 수 있음. 
- 특정 코딩 스타일, 패턴, 라이브러리 사용 등에 관한 규칙을 지정할 수 있음. 
- 팀 프로젝트에서 일관된 코드 생성을 유지하는 데 도움이 됨. 

## 6. 실습: 간단한 프로젝트 생성하기
Cursor를 활용해 간단한 프로젝트를 만들어보겠음.  웹 기반 할 일 목록(To-Do List) 애플리케이션을 생성해 보겠음. 

### 1. 프로젝트 구조 설계
1. Cursor를 열고 새 폴더를 만들어 프로젝트 디렉토리로 설정함. 
2. Composer(Ctrl+I)를 열고 다음과 같은 프롬프트를 입력함 :
   ```
   "HTML, CSS, JavaScript를 사용한 간단한 할 일 목록 웹 앱을 만들어주세요. 
   할 일 추가, 완료 표시, 삭제 기능이 있어야 함. 
   모든 코드는 별도의 파일(index.html, style.css, script.js)로 구성해주세요."
   ```
3. Composer가 생성한 파일 구조를 검토하고 Accept를 클릭하여 적용함. 

### 2. 기능 개선하기
1. 생성된 코드를 검토하고 채팅(Ctrl+L)을 사용하여 개선 사항을 요청함 :
   ```
   "할 일 항목에 우선순위를 추가하고 싶음.  
   각 항목에 높음, 중간, 낮음 우선순위를 설정할 수 있게 수정해주세요."
   ```
2. Apply 버튼을 클릭하여 제안된 변경사항을 적용함. 

### 3. 로컬 저장소 기능 추가하기
1. 채팅이나 Composer를 사용하여 로컬 저장소 기능을 추가함 :
   ```
   "localStorage를 사용하여 할 일 목록이 브라우저에 저장되도록 구현해주세요.
   페이지를 새로고침해도 데이터가 유지되어야 함. "
   ```
2. 제안된 코드를 검토하고 적용함. 

### 4. 실행 및 테스트
1. 터미널을 열고 간단한 HTTP 서버를 실행함 :
   ```
   "로컬 웹 서버를 실행하는 명령어를 알려주세요."
   ```
2. AI가 제안하는 명령어(예: Python의 http.server 모듈 사용)를 터미널에서 실행함. 
3. 브라우저에서 애플리케이션을 열고 테스트함. 

### 5. 디버깅 및 개선
1. 발견된 버그나 개선점을 채팅에 설명함 :
   ```
   "항목 삭제 후 localStorage가 업데이트되지 않는 문제가 있음.  수정해주세요."
   ```
2. AI의 제안에 따라 코드를 수정하고 다시 테스트함. 

# 실습

## 1. 설치 및 셋팅
1. [Cursor 공식 웹사이트](https://cursor.com)에서 설치 파일을 다운로드함. 
2. 운영체제에 맞는 버전을 설치함. 
3. 초기 실행 시:
   - VS Code 설정 가져오기 여부 선택
   - 원하는 테마 선택
   - GitHub 계정으로 로그인
   - AI 기능 활성화를 위한 API 키 설정 (필요한 경우)

## 2. Mode 수정
1. Command Palette 열기 (Cmd/Ctrl + Shift + P)
2. "Cursor: Change AI Model" 검색
3. 사용 가능한 모델 중 선택:
   - GPT-3.5
   - GPT-4
   - Claude
   - Anthropic Claude
   각 모델의 특징과 장단점을 고려하여 선택

## 3. Cursor로 간단한 Express 앱 개발
1. 새 프로젝트 폴더 생성:
   ```bash
   mkdir express-demo
   cd express-demo
   ```

2. Composer (Cmd/Ctrl + I) 사용하여 초기 설정:
   ```
   "Express.js를 사용한 기본 웹 서버를 설정해주세요. 
   package.json, app.js, 그리고 기본 라우트를 포함해주세요."
   ```

3. 기본 API 엔드포인트 추가:
   - GET /api/hello
   - POST /api/data
   채팅(Cmd/Ctrl + L)을 사용하여 각 엔드포인트 구현 요청

4. 미들웨어 추가:
   - 로깅
   - CORS
   - 에러 처리

5. 테스트 실행:
   ```bash
   npm install
   npm start
   ```

## 4. AutoRun 모드 활성화하기
1. Settings 열기 (Cmd/Ctrl + ,)
2. "Auto Run" 검색
3. 다음 설정 활성화:
   - Enable Auto Run
   - Auto Run on Save
   - Auto Run on Change
4. 자동 실행 규칙 설정:
   - 파일 유형별 설정
   - 특정 폴더/파일 제외 설정
   - 실행 간격 설정

## 5. Agent의 모델 변경
1. Command Palette에서 "Cursor: Configure Agent" 선택
2. Agent 설정 파일 (.cursorrules) 생성:
   ```json
   {
     "model": "claude-3-opus-20240229",
     "temperature": 0.7,
     "maxTokens": 2000,
     "rules": {
       "codeStyle": "standard",
       "language": "ko",
       "framework": "express"
     }
   }
   ```
3. 커스텀 프롬프트 설정:
   - 코딩 스타일 가이드
   - 선호하는 라이브러리/프레임워크
   - 문서화 규칙

## 6. MCP 설정
1. MCP(Multi-Context Protocol) 활성화:
   - Command Palette에서 "Enable MCP" 선택
   - 워크스페이스 설정에서 MCP 옵션 구성

2. MCP 기능 구성:
   - 파일 시스템 접근 권한 설정
   - 명령어 실행 권한 설정
   - 웹 검색 기능 활성화

3. MCP 사용 예시:
   - 다중 파일 동시 편집
   - 프로젝트 전체 리팩토링
   - 코드베이스 분석 및 문서화

4. 보안 설정:
   - 허용된 디렉토리 설정
   - API 키 및 인증 정보 관리
   - 로깅 및 모니터링 설정

## 실습 마무리
- 각 설정이 제대로 작동하는지 테스트
- 문제 발생 시 로그 확인 및 트러블슈팅
- 필요한 경우 설정 미세 조정
- 팀원들과 설정 공유 및 동기화
